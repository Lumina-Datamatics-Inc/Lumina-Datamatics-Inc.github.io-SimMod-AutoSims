SimMod uses the Vue.js single file architecture. In modern UI development, we have found that instead of dividing the codebase into three huge layers that interweaves with one another, it makes much more sense to divide them into loosely-coupled components and compose them. Inside a component, its template, logic and styles are inherently coupled, and collocating them actually makes the component more cohesive and maintainable.

CodeBlock: Template


Every activity view template must have one single root element. Our root element for every activity view template is the view-box. The next required element is the content-box. All other activity html will be placed inside here. The ViewBox and ContentBox are both covered in detail in the Widgets section.

CodeBlock: Script


The script block is made up of a list of imports and a default class object. The class object contains properties and methods that are required for the view to function properly.

Line 2, 3, 4: import statements. You use them to include external script files and widgets into the view.

Line 5: export object, this is the default class object that defines our view.

Line 6: name property, it is used for registering the view component. Uppercase syntax is required.

Line 7: components property, it is an object that contains each of the widgets that you import.

Line 11: data property method. It returns an object of data properties.

Line 15: mounted life-cycle hook method. We place view startup code here.

Line 18: methods property. Contains helper methods for the view.
