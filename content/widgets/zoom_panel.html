The ZoomPanel widget is used to provide zooming to an activity to the content-box. So you must place it directly after the opening tag as shown below. You then place any elements inside the zoom-panel that you want to be zoomed. If you have elements that you do not want to zoom, then place them outside the zoom-panel as shown below.

The zoom-panel provides two levels of zooming in and out.

The zoom-panel requires a background image, so you need to add a data property object named zoomPanelBgImg, you are passing in two css properties, the background and backgroundSize. The background requires a full path to the image. All zoom-panel background images need to be located at '/assets/img/zoom-panel/engine_compartment_zoom_bg.png'.

The background image for the zoom-panel must be exactly 2560 x 1440. If the image is not this size the background will not align properly when zoomed. The backgroundSize needs to be set to 105%.

The next thing that you need to do is to show and enable the Zoom Icons.



All icons are part of the $core.IconBar object, and Zoom is an object within that. Zoom has four methods.
  1. show()
  2. enable()
  3. disable()
  4. hide()
show() - simply displays the icons on the iconbar, at this point that are disabled and can not be clicked. They will be grayed out.
enable() - this makes the icons clickable, and changes them to white.
disable() - this returns the icons to their default state, and they turn gray again.
hide() - this removes them from the iconbar.

When you use the zoom-panel you will need to add a css class for the content-box and set it's overflow property to hidden. You do not need to add a class attribute to the content-box, the widget already has it defined.
